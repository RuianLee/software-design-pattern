## 類別圖大重點
1. 透过靜態結構，可以重复被實體化，使循環的雜訊被消除
    - 消滅了循環的雜訊，專注在單次的靜態架構
    - 一次流程的「封閉生態圈」，就是識別判斷的 context

2. 看需求分成两種「推動流程文字」，與「推動流程文字」
    - 推動流程文字（橘）: 流程時機、流程時機、事件順序
        - EX: 在第一回合中，由擁有梅花3的玩家首先出牌。
        - 第一 => 流程時機, 擁有梅花3的首先出牌 => 流程時機
        - 事件順序: 把多個單句分析的結果，順成一個流程

    - 結構行為文字（黑）: 在流程循環中定義角色、行為、關係或約束

3. 將需求視為一個 context 流程
    - 「結構行為文字」: 單次流程參與的角色，角色行為賦能，完善角色之間的關係 ＝> 使靜態關係重複被實體化
    - 「推動流程文字」, 將靜態架構的元素擺放整齊，直到完成功能需求

4. 一個大的 Context（例如：整個遊戲），內含許多小的 Context
    - 通常會有一個大類別職責裡面，裝主要流程的 Context 通常在一個類別的就好
